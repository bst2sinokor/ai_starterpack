추가 기능

Microsoft Excel 안에서 Claude를 바로 사용할 수 있는 공식 추가 기능(Add-in). 시트를 떠나지 않고 사이드 패널에서 데이터 가공·수식 작성·보고서 생성을 모두 처리할 수 있어요.

Claude for Excel 설치 방법
1
Microsoft Excel을 엽니다. Microsoft 365 구독이 있어야 사용 가능하며, Windows·Mac 데스크톱과 Excel for the Web 모두 동일하게 작동해요.
2
리본 메뉴에서 삽입(Insert) 탭을 클릭한 뒤 추가 기능 가져오기(Get Add-ins)를 선택하세요. (한국어 환경에서는 삽입 → 추가 기능)
3
Office 추가 기능 창의 검색창에 Claude를 입력하면 Claude for Excel (by Anthropic)이 나옵니다. 추가(Add) 버튼을 클릭하세요.
4
설치가 끝나면 우측에 Claude 사이드 패널이 자동으로 열립니다. Sign in을 누르고 평소 사용하는 Anthropic(Claude) 계정으로 로그인하면 끝!
5
분석할 셀 범위를 시트에서 드래그한 뒤 사이드 패널에 자연어로 지시하세요. "선택한 범위에서 월별 합계를 새 시트에 정리해줘"처럼요.

2 Chat — AI와 대화하기

가장 기본적인 모드. 자연어로 질문하고 답변을 받는 AI 대화 공간입니다. 문서 분석, 번역, 작성, 요약 등 대부분의 업무를 여기서 처리할 수 있어요.

⚙ 주요 특징
  • 📄 파일 첨부: PDF·Word·Excel·이미지를 드래그&드롭으로 바로 첨부
  • 🔗 링크 분석: URL을 붙여넣으면 해당 페이지 내용을 분석
  • 🎨 Artifacts: 코드·표·다이어그램 등 결과물을 오른쪽 패널에 렌더링
  • 💾 대화 저장: 모든 대화는 왼쪽 사이드바에 자동 저장
🖱 사용 방법
  1. 새 대화 시작: 좌측 사이드바 상단 + New chat 클릭 → 오른쪽 입력창에 질문 입력
  2. 파일 첨부: 입력창 왼쪽 + 버튼 클릭 또는 파일을 입력창에 직접 드래그&드롭. 첨부 후 질문 입력
  3. 전송: Enter 또는 오른쪽 화살표 버튼 클릭. 답변이 생성되는 동안 기다리세요
  4. Artifacts 활용: 코드·표 등이 생성되면 오른쪽 패널에 렌더링됨. 복사·다운로드 버튼으로 바로 추출 가능
  5. 대화 이어가기: 답변이 마음에 들지 않으면 추가 요청 가능. "더 간결하게", "표로 정리해줘" 처럼 후속 지시 입력
  6. 이전 대화 찾기: 좌측 사이드바 Recents에서 이전 대화 클릭하면 이어서 진행 가능
💡 이렇게 활용하세요
  • 📊 계약서 PDF 첨부 → "이 계약서의 핵심 조항과 리스크 요인을 요약해줘"
  • ✉️ 영문 이메일 작성 → "다음 내용을 격식 있는 영문 이메일로 작성해줘"
  • 🌐 해외 문서 번역 → "이 선화증권(B/L)을 한국어로 번역해줘"
  • 📈 데이터 해석 → "이 엑셀 데이터에서 이상치와 트렌드를 분석해줘"
💡 역할 지정 팁: "너는 10년 경력의 해운 물류 전문가야." 처럼 역할을 먼저 지정하면 훨씬 전문적인 답변이 나와요!

3 Cowork — AI가 직접 일하기

Chat 보다 한 단계 위의 모드. Claude가 직접 파일을 열고, 수정하고, 앱을 제어하며 작업을 완료합니다. 지시만 내리면 Claude가 알아서 처리해요.

⚙ 주요 특징
  • 📁 로컬 파일 제어: 폴더 정리, 파일 이동·생성·수정을 직접 수행
  • 🌐 앱 연동: Slack·Gmail·Google Drive 등 외부 서비스와 연결
  • ⏰ 예약 작업: "매주 금요일마다 주간 보고서 정리해줘" 같은 반복 작업 설정
  • ✅ 단계별 승인: 중요한 작업 전에 반드시 사람에게 확인 요청
🖱 사용 방법
  1. 작업 폴더 설정 (필수 첫 단계): 입력 창 하단 프로젝트에서 작업 → 폴더 선택... 또는 폴더 선택... 버튼 클릭 → Claude가 접근할 로컬 폴더를 지정합니다. 이 폴더 안의 파일만 Claude가 읽고 수정할 수 있어요. 처음엔 테스트용 빈 폴더를 새로 만들어 지정하는 것을 권장합니다. 실수로 중요한 파일이 변경되는 것을 방지할 수 있습니다.
  2. 작업 지시: 하단 입력창에 원하는 작업을 구체적으로 입력합니다. "Downloads 폴더를 날짜별로 정리해줘" 처럼 명확하게 지시할수록 결과가 좋아요.
  3. 계획 확인 및 승인: Claude가 작업 계획을 먼저 보여줍니다. 내용을 검토한 뒤 승인 버튼을 눌러야 실제 작업이 시작됩니다. 마음에 들지 않으면 수정 요청도 가능해요.
  4. 실시간 진행 확인: 작업이 진행되는 동안 각 단계를 실시간으로 볼 수 있습니다. 중간에 멈추고 싶으면 언제든 중단 버튼 클릭.
  5. 앱 연동 설정: 좌측 사이드바 Customize 클릭 → Connectors 탭에서 Slack·Gmail·Drive 등 연결할 서비스를 추가합니다.
  6. 반복 작업 예약: 입력창 하단 Scheduled 메뉴에서 작업 주기(매일/매주 등)와 시간을 설정하면 자동 반복 실행됩니다.
💡 이렇게 활용하세요
  • 🗂️ "Downloads 폴더를 종류별로 자동 정리해줘"
  • 📊 "슬랙 메시지와 이메일에서 오늘의 업무 우선순위를 뽑아 브리핑해줘"
  • 📋 "이 CSV 데이터로 주간 실적 보고서 엑셀 파일을 만들어줘"
  • 🔄 "매일 아침 9시에 뉴스 요약본을 만들어 Slack에 올려줘"
💡 주의: Cowork는 실제 파일을 수정합니다. 처음엔 테스트 폴더에서 먼저 시도해보세요. 매 단계마다 Claude가 승인을 요청하니 안심하고 사용할 수 있어요.

4 Code — AI 코딩 에이전트

코딩 경험이 없어도 됩니다! 원하는 것을 말로 설명하면 Claude가 직접 코드를 작성·수정·실행하고 결과를 보여줍니다.

⚙ 주요 특징
  • 💻 터미널 연동: 코드를 작성하고 바로 실행해서 결과 확인
  • 🔧 Antigravity 연동: 편집 중인 파일을 Claude가 직접 수정
  • 🐛 자동 디버깅: 오류가 나면 스스로 원인을 분석하고 수정
  • 📦 프로젝트 이해: 전체 폴더 구조를 파악해서 일관성 있게 작업
🖱 사용 방법
  1. 작업 폴더 설정 (필수 첫 단계): 화면 하단 폴더 선택... 버튼 클릭 → 작업할 프로젝트 폴더를 지정합니다. Claude는 이 폴더 안의 파일만 읽고 수정할 수 있어요. 폴더를 지정해야 Claude가 전체 프로젝트 구조를 파악하고 일관성 있는 코드를 작성할 수 있습니다. 처음엔 새 빈 폴더를 만들어 지정하는 걸 권장합니다.
  2. 작업 지시: 하단 입력창에 만들고 싶은 것을 자연어로 설명합니다. 코딩 지식이 없어도 돼요. "엑셀 파일을 읽어서 월별 합계를 그래프로 그려줘" 처럼 결과 위주로 설명하면 됩니다.
  3. 진행 과정 확인: Claude가 어떤 파일을 만들고 어떤 명령을 실행하는지 단계별로 보여줍니다. 각 단계마다 승인 여부를 물어보니 원치 않으면 멈출 수 있어요.
  4. 오류 발생 시: 코드 실행 중 오류가 나면 Claude가 자동으로 원인을 분석하고 수정합니다. 별도로 오류를 붙여넣거나 설명하지 않아도 됩니다.
  5. Antigravity 연동: Antigravity에서 해당 폴더를 열면 Claude가 수정한 파일을 바로 확인하고 편집할 수 있습니다. Claude Code 확장을 설치하면 Antigravity 안에서도 직접 지시가 가능해요.
  6. 결과 확인: 완성된 코드는 지정 폴더에 저장됩니다. 웹 페이지라면 Live Server로, Python이라면 터미널에서 바로 실행해 결과를 확인하세요.
💡 이렇게 활용하세요
  • 📊 "엑셀 데이터를 읽어서 그래프로 시각화하는 Python 코드 만들어줘"
  • 🌐 "우리 팀 업무 현황을 보여주는 간단한 웹 대시보드 만들어줘"
  • 🤖 "매일 환율 데이터를 수집해서 이메일로 보내주는 자동화 스크립트 만들어줘"
  • 🔍 "이 코드에서 버그를 찾아서 고쳐줘"
💡 초보자 팁: "파이썬을 모르는 비개발자인데, 이런 기능이 필요해." 처럼 본인 수준을 먼저 알려주면 Claude가 설명도 같이 해줍니다!

1 설치 및 시작

Google이 VS Code를 기반으로 만든 AI 코드 에디터. Gemini AI가 기본 내장되어 있고, Google 계정 하나로 설치부터 AI 코딩까지 바로 시작할 수 있어요.

⚙ 주요 특징
  • 🤖 Gemini Code Assist 기본 탑재: 별도 설치 없이 설치 즉시 AI 코딩 가능
  • 🆓 Google 계정으로 무료 시작: 월 무료 사용량 제공, 신용카드 불필요
  • 🔌 VS Code 완전 호환: 기존 VS Code 확장 프로그램 그대로 사용 가능
  • 🌐 멀티 AI 지원: Gemini 외 Claude, GPT 등 다른 AI 모델도 확장으로 추가 가능
🖱 사용 방법
  1. 다운로드: antigravity.google 접속 → 운영체제에 맞는 설치 파일 다운로드
  2. 설치: 다운로드된 파일 실행 → 기본 설정 그대로 Next → Install
  3. Google 로그인: 첫 실행 시 Google 계정으로 로그인 클릭 → 브라우저에서 인증 완료
  4. 한국어 설정 (선택): Ctrl+Shift+X → Korean Language Pack 검색 → 설치 → 재시작
  5. 시작 확인: 왼쪽 사이드바에 Gemini 아이콘이 보이면 AI 코딩 준비 완료!
💡 이렇게 활용하세요
  • 💻 코딩 경험이 없어도 OK — Gemini에게 "이런 프로그램 만들어줘"라고 말로 설명하면 됩니다

🖱 사용 방법
  1. 채팅 패널 열기: 왼쪽 사이드바 Gemini(별 모양) 아이콘 클릭 → 하단 채팅창에 질문 입력
  2. 코드 생성 요청: "파이썬으로 엑셀 파일을 읽어서 합계를 계산하는 코드 만들어줘" 처럼 자연어로 설명

4 Live Server — 실시간 미리보기

HTML/CSS 파일을 수정하면 저장과 동시에 브라우저가 자동 새로고침됩니다. 하나씩 새로고침할 필요 없이 수정 결과를 바로 확인할 수 있어요.

⚙ 주요 특징
  • 🔄 자동 새로고침: 파일 저장(Ctrl+S)과 동시에 브라우저 화면 즉시 갱신
  • 🌐 로컬 서버 실행: 기본 포트 5500에서 로컬 웹 서버로 파일 서빙
  • 📱 모바일 미리보기: 같은 Wi-Fi 내 스마트폰에서도 접속 가능 (IP:5500)
  • 🎨 CSS 변경 반영: HTML뿐만 아니라 CSS, JavaScript 수정도 즉시 반영
🖱 사용 방법
  1. Live Server 설치: Ctrl+Shift+X → 검색창에 Live Server 입력 → Ritwick Dey 제작 확장 설치
  2. 서버 시작: HTML 파일 우클릭 → Open with Live Server 클릭
  3. 브라우저 확인: 자동으로 기본 브라우저가 열리고 localhost:5500/파일명.html 주소로 접속
  4. 수정 → 저장 → 확인: 편집기에서 코드 수정 → Ctrl+S → 브라우저 자동 갱신 확인
  5. 서버 중지: 하단 상태바의 Port: 5500 클릭 → Stop
💡 이렇게 활용하세요
  • 🎨 이 AI 스타터팩 HTML 파일을 Live Server로 열고 수정하면서 결과를 바로바로 확인하세요
  • 📐 CSS 색상이나 여백 값을 바꾸면서 디자인 미세 조정 — 저장마다 즉시 확인 가능
  • 📱 같은 Wi-Fi의 스마트폰으로 접속해서 모바일 화면 동시 확인도 가능
⚠️ 주의 (WSL 사용자): WSL(Linux) 환경에서는 Live Server가 Windows 브라우저를 자동으로 열지 못합니다. Windows 브라우저 주소창에 http://localhost:5500/을 직접 입력해서 접속하세요.

노트북에 분석할 자료를 소스(Source)로 추가합니다. AI는 여기 추가된 자료 안에서만 답변해요.

⚙ 주요 특징
  • 📄 파일 업로드: PDF, Word, 텍스트, 오디오, 동영상 파일 지원

  • ▶️ YouTube 연동: 유튜브 링크 입력 → 자막 자동 분석 (자막 있는 영상만)
  • 📊 Google Drive 연동: Drive 문서·스프레드시트를 직접 소스로 추가 가능
🖱 사용 방법
  1. 소스 패널 열기: 노트북 화면 왼쪽 소스 패널에서 + 소스 추가 클릭
  2. 파일 업로드: 파일 업로드 선택 → PC에서 파일 선택 (PDF 권장)
  3. URL 추가: 웹사이트 선택 → 분석할 페이지 URL 붙여넣기 → 확인
  4. YouTube 추가: YouTube 선택 → 영상 URL 입력 → 자막이 자동으로 소스로 변환
  5. 소스 관리: 추가된 소스 왼쪽 체크박스로 특정 소스만 선택해 질문 가능
💡 이렇게 활용하세요
  • 📑 회사 내부 보고서 여러 개를 한 번에 업로드 → "이 자료들의 공통 결론이 뭐야?"

  • 🎬 업무 관련 유튜브 강의 URL 추가 → "핵심 내용만 정리해줘"

  5. 추천 질문 활용: 소스 추가 후 화면 하단에 AI가 자동으로 추천 질문을 제시해줌

  • 📊 "이 보고서의 핵심 인사이트 3가지만 뽑아줘" — 긴 보고서를 5초 만에 요약

1 접속 및 시작

Google의 AI 어시스턴트. Google 계정으로 별도 가입 없이 즉시 사용할 수 있고, Gmail·Drive·Calendar 등 구글 서비스와 직접 연동되는 것이 가장 큰 강점입니다.

⚙ 주요 특징
  • 🔗 Google 서비스 완전 연동: Gmail·Drive·Docs·Calendar를 AI가 직접 읽고 처리
  • 🖼️ 멀티모달: 텍스트·이미지·PDF·오디오를 동시에 이해하고 분석
  • 💎 Gems 기능: 역할·말투·형식을 미리 설정한 나만의 AI를 여러 개 만들어 저장 가능
  • 🔍 Google 검색 연동: 실시간 웹 검색 결과를 바탕으로 최신 정보 답변
🖱 사용 방법
  1. 접속: 브라우저에서 gemini.google.com 입력 → Google 계정 자동 로그인
  2. 대화 시작: 하단 입력창 클릭 → 질문 입력 → Enter 또는 전송 버튼
  3. 파일 첨부: 입력창 왼쪽 📎 아이콘 → 이미지·PDF 파일 선택 후 질문 입력
  4. 이전 대화 확인: 왼쪽 사이드바에서 과거 대화 이력 검색 및 이어서 진행 가능
  5. 모델 확인: 화면 상단에서 현재 사용 중인 Gemini 모델 버전 확인 가능
💡 이렇게 활용하세요
  • 📧 "이 내용으로 격식 있는 영문 비즈니스 이메일을 작성해줘" — 영문 메일 초안 5초 완성
  • 🌐 "오늘 해운 운임 관련 최신 뉴스 요약해줘" — Google 검색 연동으로 실시간 정보 수집
  • 📸 차트 이미지 첨부 → "이 그래프의 핵심 트렌드를 설명해줘" — 시각 자료 즉시 분석
⚠️ 주의: 무료 플랜과 유료 Advanced 플랜은 사용 모델과 기능이 다릅니다. 무료로도 일반 업무에 충분하지만, Gems 고급 기능이나 대용량 파일 처리는 Advanced에서 더 잘 작동합니다.

2 Google 워크스페이스 연동

Gemini의 가장 강력한 차별점. Gmail·Drive·Docs·Calendar를 AI가 직접 접근해 읽고 분석합니다. 다른 AI는 할 수 없는 구글 생태계 완전 통합 기능이에요.

⚙ 주요 특징
  • 📧 Gmail 연동: 받은 메일 요약, 특정 내용 검색, 답장 초안 작성
  • 📁 Drive 연동: 파일 검색, 문서 내용 요약, 파일 위치 안내
  • 📅 Calendar 연동: 오늘 일정 확인, 회의 준비 사항 정리
  • 📝 Docs 연동: 문서 내용 편집·보완·형식 변환 요청 가능
🖱 사용 방법
  1. 연동 활성화: Gemini 화면 우측 상단 설정(톱니바퀴) → Extensions → Google Workspace 토글 켜기
  2. Gmail 질문: "지난주 받은 미결 이메일 중 중요한 것 요약해줘" 입력
  3. Drive 검색: "내 Drive에서 '계약서' 관련 파일 찾아줘" 입력
  4. Calendar 확인: "오늘 내 캘린더 일정 알려줘" 입력
  5. 권한 허용: 첫 연동 시 Google 계정 접근 권한 요청 팝업 → 허용 클릭
💡 이렇게 활용하세요
  • 📬 "오늘 받은 이메일 중 답장이 필요한 것만 골라서 요약해줘" — 받은 메일함 빠른 처리
  • 📂 "Drive에서 지난달에 수정한 파일 목록 보여줘" — 작업 이력 빠르게 파악
  • 🗓️ "이번 주 회의 일정 정리하고 각 회의 준비 사항 알려줘" — 주간 업무 브리핑 자동화
⚠️ 주의: 회사 Google Workspace 계정은 IT 관리자 정책에 따라 연동이 제한될 수 있습니다. 연동이 안 되면 IT 부서에 문의하거나 개인 Gmail 계정으로 먼저 테스트해보세요.

3 이미지·파일 분석

텍스트뿐만 아니라 이미지·PDF·동영상까지 첨부해서 분석을 요청할 수 있어요. 사진 한 장만 올려도 내용을 이해하고 질문에 답합니다.

⚙ 주요 특징
  • 📷 이미지 분석: 차트·그래프·사진·스크린샷 내용을 텍스트로 설명
  • 📄 PDF 분석: 문서 첨부 후 요약·질문·번역 요청 가능
  • 🎥 동영상 분석: 짧은 동영상 업로드 시 장면 내용 파악 가능
  • 🌏 다국어 번역: 외국어 이미지(메뉴판, 서류 등)도 한국어로 번역
🖱 사용 방법
  1. 파일 첨부: 입력창 왼쪽 📎 아이콘 클릭 → 이미지·PDF 파일 선택
  2. 드래그&드롭: 파일을 Gemini 입력창으로 바로 끌어다 놓기
  3. 질문 입력: 첨부 후 원하는 내용 입력 → Enter. 첨부 파일 내용을 기반으로 답변
  4. 이미지 붙여넣기: 클립보드에 복사한 이미지(스크린샷 등)를 입력창에 Ctrl+V로 바로 붙여넣기 가능
💡 이렇게 활용하세요
  • 📊 실적 차트 스크린샷 첨부 → "이 그래프에서 주목할 만한 변화 포인트 3개 알려줘"
  • 📋 영문 계약서 PDF 첨부 → "이 계약서를 한국어로 요약해줘. 핵심 조항 위주로"
  • 🌐 외국어 메뉴판·안내문 사진 → "이게 무슨 내용이야? 한국어로 설명해줘"
⚠️ 주의: 첨부 파일에 포함된 개인정보나 기밀 정보는 Google 서버로 전송됩니다. 사내 기밀 문서나 개인정보가 담긴 파일을 첨부할 때는 사전에 회사 보안 정책을 확인하세요.

4 Gems — 나만의 전용 AI 만들기

Gems는 역할·말투·형식을 미리 설정해둔 나만의 AI입니다. 매번 "너는 이런 전문가야"라고 설명할 필요 없이 Gem을 클릭하면 바로 전문가 모드로 대화할 수 있어요.

⚙ 주요 특징
  • 🎭 역할 사전 설정: "해운 물류 전문가", "이메일 작성 도우미" 등 Gem별 역할 고정
  • 💾 여러 Gem 저장: 업무별로 다양한 Gem을 만들어 사이드바에서 빠르게 선택
  • 🔄 반복 업무 자동화: 매번 같은 형식의 결과물이 필요한 작업에 특히 효과적
  • 🧪 Preview 테스트: 저장 전 미리보기로 Gem이 올바르게 작동하는지 확인 가능
🖱 사용 방법
  1. Gem Manager 열기: 왼쪽 사이드바 Gems 클릭 → Gem Manager 화면 열기
  2. 새 Gem 만들기: 우측 상단 + New Gem 클릭
  3. 이름 입력: 용도가 명확한 이름 입력 (예: "해운 운임 분석 전문가", "회의록 요약기")
  4. 지시사항 작성: AI 역할, 말투, 출력 형식, 주의사항을 자연어로 상세히 입력
  5. 미리보기 확인: 오른쪽 Preview 패널에서 테스트 질문 입력 → 동작 확인
  6. 저장 및 사용: 상단 Save 클릭 → 이후 사이드바 Gems 목록에서 클릭해 바로 사용
💡 이렇게 활용하세요
  • 🚢 "해운 화물 운임 전문가" Gem — 매번 역할 설명 없이 클릭 즉시 전문 분석 시작
  • ✉️ "영문 비즈니스 이메일 작성기" Gem — 내용만 입력하면 항상 같은 격식체로 작성
  • 📝 "회의록 요약기" Gem — 회의 내용을 붙여넣으면 결정사항·액션아이템 형식으로 정리
⚠️ 주의: 지시사항이 모호할수록 Gem이 일관성 없이 동작합니다. "~처럼 답해줘" 보다 "반드시 ~형식으로, ~을 포함하고, ~은 제외하여 답해줘"처럼 구체적으로 작성할수록 결과가 안정적이에요.

5 Gem 지시사항 잘 쓰는 법

Gem의 성능은 지시사항(Instructions)의 품질에 달려 있어요. 같은 Gem이어도 지시사항이 얼마나 구체적이냐에 따라 결과물 수준이 크게 달라집니다.

⚙ 주요 특징
  • 🎯 역할 정의: "너는 10년 경력의 해운 물류 전문가야" — 역할을 먼저 명확히
  • 📐 출력 형식 지정: "답변은 항상 요점 3가지 + 근거 + 결론 형식으로 작성해" — 일관된 포맷
  • 🚫 금지 사항 명시: "추측성 발언은 하지 마. 확실하지 않으면 모른다고 해" — 오류 최소화
  • 🌏 언어·말투 설정: "항상 한국어로, 격식체로 답해줘" — 언제나 동일한 톤 유지
🖱 사용 방법
  1. 역할부터 시작: "너는 [역할]이야. [경력·전문 분야]를 가지고 있어."
  2. 할 일 명시: "사용자가 [어떤 내용]을 주면 [어떻게 처리]해줘."
  3. 출력 형식 지정: "답변은 반드시 [형식]으로 작성해. 예: 제목 / 요점 3가지 / 결론"
  4. 하지 말 것 명시: "~는 절대 하지 마. ~한 경우에는 ~라고 답해."
  5. 예시 추가: 지시사항 안에 입력 예시와 기대 출력 예시를 직접 넣으면 더욱 정확
💡 이렇게 활용하세요
  • 📦 지시사항 예시: "너는 해운 운임 분석 전문가야. 운임 데이터를 주면 주요 변동 원인 3가지와 향후 전망을 표로 정리해줘. 숫자는 반드시 출처와 함께 제시해."
  • ✉️ 지시사항 예시: "사용자가 이메일 내용 요점을 주면, 이를 격식 있는 영문 비즈니스 이메일로 작성해줘. 제목(Subject)도 함께 제안해줘."
⚠️ 주의: Gem을 저장한 후 실제로 테스트해보세요. Preview에서 잘 작동해도 실제 사용에서 다르게 동작할 수 있어요. 이상한 점이 있으면 지시사항을 조금씩 수정하며 튜닝하는 과정이 필요합니다.

AI 관련 용어들, 한 번 읽으면 다 이해돼요. 클릭해서 펼쳐보세요.

AI (인공지능)
▾

컴퓨터가 사람처럼 보고, 듣고, 생각하고, 말할 수 있도록 만든 기술입니다. 사람이 수천 번 연습해서 자전거 타는 법을 익히듯, AI도 수억 개의 데이터를 학습해서 점점 똑똑해집니다. 예전에는 컴퓨터가 정해진 규칙만 따랐다면, AI는 스스로 패턴을 찾아 판단하고 행동할 수 있습니다.

예: 스팸 메일을 자동으로 걸러주는 것, 유튜브가 내 취향에 맞는 영상을 추천해주는 것, ChatGPT가 질문에 답해주는 것 모두 AI 기술입니다.
LLM (대규모 언어 모델)
▾

세상에 존재하는 책, 논문, 뉴스, 웹사이트 등 수천억 개의 문서를 통째로 학습한 AI입니다. 도서관 수억 권의 책을 전부 읽고 암기한 천재 비서라고 생각하면 됩니다. 그래서 어떤 주제로 말을 걸어도 자연스럽게 대화하고, 글을 쓰고, 번역하고, 요약할 수 있습니다.

예: ChatGPT에게 "해운업계 트렌드를 요약해줘"라고 하면 척척 답하는 이유가 바로 LLM이 방대한 텍스트를 학습했기 때문입니다.
프롬프트 (Prompt)
▾

AI에게 내리는 지시문 또는 질문입니다. 같은 AI라도 어떻게 말하느냐에 따라 결과물의 품질이 완전히 달라집니다. 마치 새 직원에게 업무를 맡길 때 "보고서 써"보다 "3페이지 분량으로 경쟁사 현황을 정리하고 시사점을 포함해서 써줘"가 훨씬 좋은 결과를 내는 것과 같습니다. AI를 잘 쓰는 핵심은 결국 프롬프트를 얼마나 구체적으로 쓰느냐입니다.

예: "번역해줘" (❌) → "다음 영문 계약서를 법률 용어를 살려서 한국어로 번역하고, 이해하기 어려운 조항은 쉽게 풀어서 설명해줘" (✅)
API / API Key
▾

API는 서로 다른 프로그램이 대화할 수 있게 해주는 연결 통로입니다. 식당에 비유하면, 손님(내 앱)이 주방(AI)에 직접 들어가는 대신 종업원(API)을 통해 주문을 전달하는 것과 같습니다. API Key는 그 식당을 이용할 수 있는 회원증입니다. Key가 있어야만 AI 기능을 내 서비스에 연결할 수 있고, 없으면 접근이 차단됩니다.

예: Claude API Key를 발급받아 사내 업무 시스템에 연결하면, 직원들이 Claude를 직접 사이트에서 쓰지 않아도 우리 시스템 안에서 AI 기능을 사용할 수 있게 됩니다.
JSON
▾

컴퓨터가 데이터를 정리해서 주고받을 때 사용하는 표준 형식입니다. 사람으로 치면 엑셀 표와 비슷하지만, 컴퓨터가 훨씬 빠르게 읽고 처리할 수 있습니다. AI 설정 파일, API 응답 등에서 자주 등장하며, 처음엔 낯설어 보여도 구조 자체는 매우 단순합니다. 직접 작성할 일은 거의 없고 AI가 자동으로 생성해줍니다.

예: AI에게 "직원 정보를 JSON으로 정리해줘"라고 하면 → {"이름": "홍길동", "부서": "영업기획팀", "입사연도": 2020} 형태로 깔끔하게 만들어줍니다.
Markdown
▾

키보드 특수기호 몇 가지로 문서 서식을 표현하는 방법입니다. 워드처럼 버튼을 클릭하는 대신, 별표(**)나 샵(#) 같은 기호를 텍스트 앞뒤에 붙여서 굵게·제목·목록 등을 표현합니다. Claude나 ChatGPT가 답변할 때 자동으로 마크다운을 사용하는데, 그래서 AI 답변이 제목·목록·표 등으로 깔끔하게 보이는 겁니다. 직접 쓸 줄 몰라도 되지만, 알면 AI와 소통이 더 편해집니다.

예: AI 답변에서 **굵은 글씨**, ## 큰 제목, - 항목 나열처럼 보이는 것들이 모두 마크다운입니다. Notion, GitHub에서도 동일하게 작동합니다.
토큰 (Token)
▾

AI가 글을 읽고 처리하는 가장 작은 단위입니다. 우리가 글을 읽을 때 글자 단위로 읽듯, AI는 토큰 단위로 글을 인식합니다. AI 서비스 요금은 이 토큰 수를 기준으로 계산됩니다. 마치 택시 미터기가 거리만큼 요금을 올리듯, AI는 처리한 토큰만큼 비용이 발생합니다. 그래서 프롬프트를 너무 길게 쓰면 비용이 올라갑니다.

예: "안녕하세요, 오늘 날씨가 좋네요"라는 문장은 약 15토큰. Claude API로 이 문장을 처리하면 15토큰치 요금이 부과됩니다. 긴 보고서를 분석시키면 수천~수만 토큰이 사용됩니다.
컨텍스트 윈도우
▾

AI가 한 번의 대화에서 기억할 수 있는 최대 정보량입니다. 사람도 한 번에 너무 많은 걸 들으면 앞에서 들은 내용을 잊어버리듯, AI도 처리할 수 있는 텍스트 양에 한계가 있습니다. 이 한계를 컨텍스트 윈도우라고 합니다. 창이 클수록 두꺼운 보고서도 통째로 분석할 수 있고, 긴 대화도 처음부터 끝까지 기억합니다.

예: 컨텍스트 윈도우가 작은 AI에게 100페이지 계약서를 분석시키면 앞부분을 잊어버립니다. Claude는 200K 토큰(약 500페이지 분량)을 한 번에 처리할 수 있어서 두꺼운 문서 분석에 유리합니다.
에이전트 (Agent)
▾

단순히 질문에 답하는 AI를 넘어, 스스로 계획을 세우고 행동하는 AI입니다. 일반 챗봇은 말로만 대답하지만, 에이전트는 실제로 인터넷을 검색하고, 파일을 만들고, 코드를 실행하는 등 사람처럼 일을 처리합니다. 마치 지시만 내리면 알아서 일 처리하고 결과를 가져오는 유능한 직원과 같습니다.

예: "우리 회사 경쟁사 3곳의 최근 동향을 조사해서 보고서로 만들어줘"라고 하면 — 에이전트가 스스로 웹 검색 → 자료 수집 → 내용 정리 → 보고서 작성까지 완료합니다.
서브에이전트 (Subagent)
▾

큰 프로젝트를 팀원들에게 나눠서 처리할 때처럼, 메인 AI(팀장)가 세부 작업들을 여러 하위 AI(팀원)에게 나눠 맡기는 구조입니다. 각 서브에이전트는 자신이 맡은 일만 집중해서 처리하고 결과를 팀장에게 보고합니다. 이 방식으로 복잡한 작업을 빠르고 정확하게 완료할 수 있습니다.

예: "신규사업 기획안 만들어줘"라는 지시를 받은 메인 AI가 → 시장조사 서브에이전트, 재무분석 서브에이전트, 문서작성 서브에이전트에게 동시에 일을 나눠주고 결과를 합쳐 완성합니다.
오케스트레이터
▾

여러 AI 에이전트(팀원)를 총괄 지휘하는 AI 팀장입니다. 오케스트라 지휘자가 바이올린·첼로·피아노 각 파트에 지시를 내리고 전체 음악을 조율하듯, 오케스트레이터는 큰 목표를 작은 단위로 나눠 각 서브에이전트에게 배분하고, 결과를 모아 최종 결과물을 완성합니다. 사람이 직접 하나하나 지시하지 않아도 알아서 흐름을 관리합니다.

예: "분기 보고서 작성"이라는 큰 목표를 받은 오케스트레이터가 → 데이터 수집 에이전트 / 차트 생성 에이전트 / 문서 작성 에이전트를 차례로 또는 동시에 지휘해 보고서 한 편을 완성합니다.
MCP (Model Context Protocol)
▾

AI가 외부 프로그램이나 서비스를 사용할 수 있게 연결해주는 표준 규격입니다. USB 포트에 비유할 수 있습니다. USB가 어떤 기기든 꽂으면 작동하듯, MCP 규격을 따르면 AI가 다양한 도구(파일, 인터넷, 회사 데이터베이스 등)를 표준화된 방식으로 연결해 사용할 수 있습니다. 한 번 설정해두면 AI의 능력이 크게 확장됩니다.

예: Claude Desktop에 MCP로 회사 문서 폴더를 연결하면, "지난달 계약서 불러와서 요약해줘"라고만 해도 AI가 직접 폴더에서 파일을 찾아 읽어올 수 있습니다.
스킬 (Skill)
▾

AI에게 특정 업무를 처리하는 방법을 레시피처럼 저장해두는 기능입니다. 요리사가 레시피 북에 자주 만드는 요리 방법을 적어두듯, AI에게도 자주 반복하는 업무의 방법을 저장해두면 매번 길게 설명하지 않아도 됩니다. "스킬 사용해"라는 말 한 마디로 AI가 저장된 방식대로 완벽하게 처리합니다.

예: "주간 업무 보고서 스킬"을 만들어두면, 매주 "주간 보고서 작성해줘"라고만 해도 형식·순서·말투까지 항상 동일하게 작성됩니다.
환각 (Hallucination)
▾

AI가 모르는 것을 마치 아는 것처럼 자신 있게 틀린 답변을 하는 현상입니다. 마치 모르는 것도 당당하게 대답하는 신입사원처럼, AI도 정확한 정보가 없을 때 그럴듯하게 꾸며낸 답을 만들어냅니다. AI는 "모르겠습니다"보다는 그럴싸한 답을 만드는 쪽으로 학습되어 있어서 이런 문제가 생깁니다. 특히 숫자·날짜·법령·인명 등 구체적인 사실에서 자주 발생합니다.

예: "이 계약서 조항의 법적 근거를 찾아줘"라고 했을 때, AI가 실제로 없는 법 조항을 마치 있는 것처럼 번호까지 만들어 인용할 수 있습니다. 중요한 사실은 반드시 원문 확인이 필요합니다.
시스템 프롬프트
▾

대화가 시작되기 전에 AI에게 미리 주는 역할·규칙·성격 설정서입니다. 회사에서 신입사원에게 "우리 회사는 이런 곳이고, 이렇게 일해줘"라고 사전 교육을 하는 것과 같습니다. 시스템 프롬프트를 잘 설정하면 AI가 매번 같은 성격과 방식으로 일관되게 답변합니다. 일반 사용자에게는 보이지 않지만, 서비스 뒤에서 AI의 행동을 조종하는 핵심 설정입니다.

예: 고객 상담 AI에 "너는 Sinokor 해운의 친절한 상담사야. 항상 존댓말을 쓰고, 모르는 질문은 담당자 연결을 안내해"라고 시스템 프롬프트를 설정하면 → 이후 모든 대화에서 AI가 그 역할을 유지합니다.
RAG (검색 증강 생성)
▾

AI가 답변하기 전에 미리 저장된 문서나 자료를 검색해서 참고하는 방식입니다. 오픈북 시험에 비유할 수 있습니다. 일반 AI는 학습한 내용만으로 답하지만(암기 시험), RAG를 적용하면 우리 회사 자료를 실시간으로 찾아보고(오픈북) 정확하게 답합니다. 회사 내부 문서, 최신 자료, 개인화된 데이터를 AI에 연결할 때 핵심적으로 사용되는 기술입니다.

예: 사내 취업규칙·복리후생 문서를 RAG로 연결하면 → "우리 회사 연차 계산 방법이 어떻게 돼?" 같은 질문에 AI가 인사규정을 직접 찾아 정확히 답변합니다. 기존 AI라면 일반적인 답만 했을 것입니다.
파인튜닝 (Fine-tuning)
▾

기존 AI 모델을 우리 회사나 특정 분야에 맞게 추가로 재교육시키는 과정입니다. 요리학교를 졸업한 요리사(기본 AI)를 우리 식당에 고용해서 우리만의 레시피와 서비스 방식을 별도로 교육하는 것과 같습니다. 파인튜닝된 AI는 일반 AI보다 우리 분야 용어·형식·스타일을 훨씬 자연스럽게 처리합니다. 다만 시간과 비용이 필요해서 신중하게 결정해야 합니다.

예: 수천 건의 해운 B/L(선화증권), 화물 계약서, 운임 견적서로 AI를 파인튜닝하면 → 해운 업계 전문 용어를 정확히 이해하고 업무 서류를 우리 회사 양식에 맞게 자동으로 작성하는 특화 AI가 만들어집니다.

잘 쓰는 법

AI는 질문을 어떻게 하느냐에 따라 완전히 다른 결과를 냅니다. 이것만 알아도 AI 활용 능력이 2배 이상 올라갑니다.

이렇게 하면 안 돼요
"보고서 써줘"
어떤 보고서인지, 누구를 위한 건지, 분량은 얼마나 되는지 아무 정보가 없어서 AI가 일반적인 내용만 생성합니다.
이렇게 해보세요
"해운 물류 업계의 2024년 AI 도입 현황을 조사해서, 경영진 보고용 A4 2페이지 분량의 보고서로 작성해줘. 요약, 주요 트렌드, 시사점 순서로 구성하고, 전문적이지만 이해하기 쉬운 말투로 써줘."
대상, 분량, 구성 순서, 말투까지 명확히 지정했기 때문에 원하는 결과에 훨씬 가깝게 나옵니다.

역할 지정하기

"너는 10년 경력의 해운 전문가야"처럼 AI에게 역할을 주면 그에 맞는 전문성 있는 답변을 합니다. "전문가로서", "초보자에게 설명하듯" 등의 표현이 효과적입니다.

형식 지정하기

표로 정리해줘, 번호 목록으로 써줘, 한 문장으로 요약해줘 등 출력 형식을 명확히 하면 바로 쓸 수 있는 결과물이 나옵니다.

단계별 요청하기

복잡한 작업은 한 번에 다 요청하지 말고 단계별로 나눠서 요청하세요. 먼저 개요를 잡고, 각 섹션을 하나씩 작성하는 방식이 훨씬 좋은 결과를 냅니다.

분량 지정하기

"200자 이내로", "A4 한 페이지 분량으로", "핵심만 3줄로" 등 원하는 분량을 알려주면 너무 길거나 짧은 답변을 방지할 수 있습니다.

예시 제공하기

"이런 스타일로 써줘 [예시]"처럼 참고할 예시를 함께 제공하면 AI가 원하는 스타일을 훨씬 정확하게 파악합니다. 특히 글쓰기 작업에서 매우 효과적입니다.

피드백 주고 수정하기

AI가 처음부터 완벽하길 기대하지 마세요. "이 부분을 더 구체적으로", "3번 항목을 다르게 써줘"처럼 피드백을 주고 반복해서 개선하는 것이 핵심입니다.

Claude와 ChatGPT, 뭐가 달라요? 뭘 써야 하나요?+

Claude는 긴 문서 분석, 꼼꼼한 논리적 답변, 코딩에 강합니다. 보고서 작성, 계약서 검토, 복잡한 분석 작업에 추천합니다.

ChatGPT는 가장 많은 사용자가 있고 플러그인 생태계가 풍부합니다. 일상적인 질문, 아이디어 발산, 이미지 생성(DALL-E)에 좋습니다.

처음엔 두 개 모두 무료로 써보고 본인에게 맞는 걸 선택하는 것을 추천합니다!

유료 결제를 꼭 해야 하나요?+

처음엔 무료 플랜으로 충분합니다. Claude, ChatGPT 모두 무료로도 상당히 많은 기능을 사용할 수 있습니다.

다만 하루 사용량 제한이 있어서 자주 쓰다 보면 막히는 경우가 생깁니다. 업무에 적극적으로 활용할 계획이라면 월 $20(약 2만 8천원)의 Pro 플랜이 충분히 가치 있습니다.

회사 기밀 정보를 AI에 입력해도 안전한가요?+

주의가 필요합니다. Claude, ChatGPT 등 대부분의 AI는 입력된 내용을 서비스 개선을 위해 활용할 수 있습니다.

기밀 정보나 개인정보는 입력하지 않는 것이 원칙입니다. 민감한 내용은 익명화하거나 일반화해서 입력하세요.

기업용 플랜(Claude for Teams, ChatGPT Enterprise)은 데이터를 학습에 사용하지 않아 보안이 강화됩니다.

AI가 틀린 말을 하면 어떡하죠?+

AI는 환각(Hallucination) 현상이 있어서 가끔 틀린 정보를 자신 있게 말하기도 합니다.

중요한 수치, 법령, 날짜, 인용 등은 반드시 직접 확인하는 습관이 필요합니다. AI 답변은 "초안" 또는 "참고 자료"로 활용하고, 최종 확인은 사람이 하는 것이 원칙입니다.

최근에는 Perplexity처럼 출처를 함께 제공하는 AI 도구를 쓰면 검증이 수월합니다.

코딩을 전혀 모르는데 Claude Code를 쓸 수 있나요?+

네, 사용 가능합니다! Claude Code는 자연어(한국어)로 지시하면 AI가 알아서 코드를 작성하고 실행합니다. 코딩 지식이 없어도 원하는 것을 말로 설명하면 됩니다.

다만 Node.js 설치 등 기본 환경 세팅은 필요합니다. 처음엔 Claude Desktop으로 시작해서, 익숙해지면 Claude Code를 시도해보는 것을 추천드립니다.

처음엔 뭐부터 시작하면 되나요?+

이 순서를 추천합니다!

1단계: 구글 계정으로 Claude 또는 ChatGPT 무료 가입
2단계: 업무에서 반복적으로 하는 작업(이메일 작성, 번역, 요약 등)을 AI로 해보기
3단계: NotebookLM으로 PDF 보고서 분석 시도
4단계: Gamma로 AI 프레젠테이션 자동 생성 시도
5단계: 익숙해지면 Claude Desktop 설치 → Claude Code 도전!

처음부터 모든 걸 배우려 하지 말고, 당장 내 업무에 쓸 수 있는 것 하나부터 시작하세요!

AI가 제 일자리를 빼앗아가지 않나요?+

AI는 사람을 대체하기보다 업무 생산성을 높여주는 도구입니다. 반복적인 단순 업무를 AI가 맡고, 사람은 더 창의적이고 전략적인 일에 집중할 수 있게 됩니다.

지금 시점에서 중요한 건 "AI를 쓰는 사람"과 "AI를 모르는 사람"의 격차입니다. AI를 잘 활용하는 것 자체가 앞으로의 핵심 역량이 될 것입니다. 먼저 배우고 먼저 활용하는 것이 답입니다!
